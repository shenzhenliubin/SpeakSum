# SpeakSum 产品设计文档

**文档版本**: 2.0  
**更新日期**: 2026-04-06  
**状态**: REDESIGNED

---

## 1. 产品定位

SpeakSum 不再是“会议纪要工具”，而是一个只服务 **刘彬本人** 的个人思想整理系统。

它接收两类输入：
- **会议纪要**：从多人会议中筛出刘彬的发言，再沉淀为思想记录
- **其他文本**：默认整份内容就是刘彬本人写下或表达的内容

系统的核心产出只有两类：
- **发言总结**：对这次输入内容中刘彬核心表达的整体总结
- **思想金句**：具有明确观点、洞察、认知或方法的高价值思想单元

知识图谱不再围绕“单条发言”或“话题岛屿”构建，而是围绕：
- **领域（Domain）**
- **思想金句（Quote）**

**核心价值主张**
> 让每次会议发言、每段随笔和每篇文章，都沉淀为刘彬自己的思想资产。

---

## 2. 产品原则

### 2.1 固定主身份

- 产品默认服务对象就是 **刘彬**
- 不再把“是否是刘彬”作为所有来源的统一前提
- 只有在 `会议纪要` 来源下，系统才需要判断并筛出刘彬发言
- 在 `其他文本` 来源下，系统默认整份内容都属于刘彬本人

### 2.2 结果优先级

每条内容记录的结果层级固定为：
1. **发言总结**：一段结构化、客观的总结文字
2. **思想金句**：3-8 条思想型金句，宁少勿滥
3. **领域归类**：每条金句关联 1-3 个领域

### 2.3 图谱原则

- 知识图谱只和 **思想金句** 挂钩
- 图谱节点核心是 **领域**
- 点开领域后，展示相关金句与来源内容
- `发言总结` 是内容详情页的主结果，但不直接进入图谱结构

---

## 3. 目标用户

当前版本只面向 **刘彬本人** 使用。

这意味着系统的设计目标不是“多用户 SaaS 平台”，而是：
- 帮助刘彬持续积累自己的发言与文字输出
- 让会议表达、文章、笔记进入同一条思想沉淀链路
- 让知识图谱服务于“长期思想回看”而不是“会议转写管理”

---

## 4. 支持的内容来源

### 4.1 会议纪要

定义：多人会议纪要、转写稿、会议记录。

系统任务：
- 理解整场会议背景
- 从多人内容中筛出刘彬发言
- 如果没有足够证据表明刘彬在发言，则标记为 `已忽略`
- 如果存在刘彬发言，则生成发言总结与思想金句

### 4.2 其他文本

定义：刘彬本人写下的文章、随笔、笔记、备忘录、长文本草稿等。

系统任务：
- 默认整份内容都属于刘彬本人
- 不再做发言人识别
- 直接生成发言总结与思想金句

---

## 5. 核心用户流程

### 5.1 上传与生成

```text
选择来源类型
  -> 上传文件
  -> 系统解析文本
  -> 按来源类型走不同处理流程
  -> 生成发言总结
  -> 提炼思想金句
  -> 给金句归类领域
  -> 更新知识图谱
  -> 进入详情页
```

### 5.2 会议纪要处理流程

```text
上传会议纪要
  -> 解析全文
  -> 识别刘彬发言
  -> 若未识别到，标记“已忽略”
  -> 若识别到，生成发言总结 + 思想金句 + 领域归类
  -> 更新知识图谱
```

### 5.3 其他文本处理流程

```text
上传其他文本
  -> 解析全文
  -> 默认视为刘彬本人输出
  -> 生成发言总结 + 思想金句 + 领域归类
  -> 更新知识图谱
```

## 6. 默认领域体系

领域使用稳定 ID，显示名可调整，但默认不鼓励频繁变更。

| 领域名称 | 领域 ID | 说明 |
|----------|----------|------|
| 产品与业务 | `product_business` | 产品判断、业务方向、用户价值、商业逻辑 |
| 技术与架构 | `technology_architecture` | 技术体系、架构设计、工程方法、系统边界 |
| 项目推进与交付 | `delivery_execution` | 目标拆解、执行推进、项目节奏、交付质量 |
| 组织协同与管理 | `organization_collaboration` | 团队协作、组织机制、职责边界、管理判断 |
| 学习成长与认知 | `learning_growth` | 学习方法、成长路径、认知升级、自我迭代 |
| 方法论与决策 | `decision_method` | 判断框架、思考方法、决策原则、分析路径 |
| 人生选择与价值观 | `life_values` | 人生观、价值排序、长期取舍、个人哲学 |
| 运动健康与身心状态 | `health_fitness` | 运动、健康、作息、精力管理、身心状态 |
| 下一代教育与成长 | `next_generation_education` | 子女教育、成长环境、教育方法、家庭影响 |
| 投资研究与交易决策 | `investing_trading` | 投资框架、交易判断、研究方法、风险认知 |
| 其他 | `other` | 不适合归入其他领域的内容 |

## 7. 页面与信息架构

### 7.1 顶层结构

```text
SpeakSum
├── 首页
├── 时间线
├── 知识图谱
├── 上传
└── 用户中心
```

### 7.2 首页

首页不再强调“会议管理”，而是强调个人思想资产：
- 最近内容记录
- 最近新增金句
- 领域分布概览
- 快速上传入口

### 7.3 时间线

时间线显示的是 **内容记录时间线**，而不是纯会议时间线。

每条卡片展示：
- 标题
- 日期
- 来源类型（会议纪要 / 其他文本）
- 发言总结摘要
- 思想金句数量
- 处理状态

### 7.4 详情页

详情页是单条内容记录的主工作区，结构固定为：
1. **发言总结**
2. **思想金句**
3. **内容信息**

其中：
- 发言总结支持编辑
- 思想金句支持编辑、删除、改领域
- 改领域会立即影响知识图谱

### 7.5 知识图谱

知识图谱页主结构：
- 图谱节点：领域
- 图谱关系：领域之间在同一内容、同一金句中的共现关系
- 侧边详情：领域下的金句列表

图谱页的核心问题不再是“这场会议里说了什么”，而是：
- 刘彬在什么领域持续输出观点
- 哪些领域经常被一起思考
- 某个领域下沉淀了哪些思想金句

## 8. 核心功能优先级

### P0

| 功能 | 描述 | 验收标准 |
|------|------|----------|
| 来源类型选择 | 上传时选择 `会议纪要 / 其他文本` | 两种来源走不同处理链路 |
| 发言总结生成 | 为每条内容生成一段结构化总结 | 内容完整、客观、可编辑 |
| 思想金句提炼 | 输出 3-8 条思想型金句 | 有明确观点/洞察，不凑数 |
| 领域归类 | 为每条金句分配 1-3 个领域 | 只允许使用预定义领域 ID |
| 知识图谱 | 按领域聚合思想金句 | 可浏览、可点击、可追溯到内容 |
| 会议纪要忽略态 | 未检测到刘彬发言时标记已忽略 | 不进入主时间线和图谱 |

### P1

| 功能 | 描述 | 验收标准 |
|------|------|----------|
| 发言总结编辑 | 手工编辑总结文字 | 单条保存 |
| 金句编辑/删除 | 修改思想金句内容 | 单条保存，单条删除 |
| 金句领域编辑 | 修改金句关联领域 | 保存后图谱立即更新 |
| 搜索与筛选 | 按标题、日期、领域搜索 | 快速定位内容 |
| 多文件上传 | 批量上传多个内容文件 | 逐条反馈状态 |

### P2

| 功能 | 描述 |
|------|------|
| 领域名称后台维护 | 允许低频修改领域显示名 |
| 周期摘要 | 按周/月输出思想变化摘要 |
| 导出 | 导出内容、金句和领域关系 |

## 9. 内容生成标准

### 9.1 发言总结

- 是对刘彬本次表达的整体客观总结
- 可以组织成“首先、其次、再次”，也可以是一段完整文字
- 不是标签列表，也不是金句拼接
- 建议长度：120-260 字

### 9.2 思想金句

- 本质是 **思想摘录**，页面名称沿用“金句”
- 必须表达明确观点、洞察、认知或方法
- 可以是短句，也可以是 1-3 句话
- 不是漂亮话，不是流程性语言，不是背景复述
- 默认输出 3-8 条，宁缺毋滥

---

## 10. 成功标准

产品成功不再用“提取了多少条发言”衡量，而用以下标准衡量：

- 一条内容能否沉淀出可信的发言总结
- 一条内容能否产出少量高质量思想金句
- 金句能否稳定归入正确领域
- 知识图谱是否能帮助刘彬回看长期思想积累

---

## 11. 设计结论

SpeakSum 2.0 的核心变化是：

- 从 **会议纪要工具** 变成 **刘彬个人思想整理系统**
- 从 **逐条发言 / 观点列表** 变成 **发言总结 + 思想金句**
- 从 **话题图谱** 变成 **领域图谱**
- 从 **是否检测到任意目标发言人** 变成 **固定服务刘彬本人**
- 从 **会议单一来源** 扩展为 **会议纪要 + 其他文本**

这套变化会同步影响后端数据模型、Prompt 设计、前端页面语义和知识图谱构建逻辑。
